    🛠️ How It Works

    • Consistent Hashing: Each prefix is hashed and routed to 1 of 3 Redis nodes using a virtual-node ring (150 vnodes / node).
    • Write-Behind: Search submissions are buffered in memory and batch-upserted to PostgreSQL every 5 s (or every 500 events).
    • Scoring: score = frequency×1 + recency_score×50 where recency_score uses exponential decay: R_new = R_old·e−λΔt + 1.
    • Debouncing: Frontend waits 300 ms after the last keystroke before calling GET /suggest.
